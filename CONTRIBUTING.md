# Contributing to Dustoff

First off, thanks for taking the time to contribute! Every contribution matters, whether it's code, bug reports, feature ideas, or documentation improvements.

## Ground Rules

- **Open an issue first.** Before starting work on a new feature or significant change, open an issue to discuss it. This avoids wasted effort and helps us coordinate.
- **Be respectful.** See our [Code of Conduct](CODE_OF_CONDUCT.md).
- **Keep PRs focused.** One logical change per PR. Don't mix unrelated fixes.

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) >= 18.18.0
- [Bun](https://bun.sh/) (used for building, testing, and package management)

### Setup

```bash
git clone https://github.com/westpoint-io/dustoff.git
cd dustoff
bun install
```

### Development Commands

```bash
bun test                    # run all tests
bun test src/features/      # unit tests only
bun test test/integration/  # integration tests only
bun run build               # build to dist/
bun run typecheck           # tsc --noEmit (must pass with zero errors)
```

### Before Submitting

1. Run `bun run typecheck` — must pass with zero errors
2. Run `bun test` — all tests must pass
3. If you added new functionality, add tests for it

## Submitting a Pull Request

1. Fork the repo and create a branch from `main`
2. Use a descriptive branch name: `feat/sort-by-age`, `fix/scan-symlinks`, `docs/update-readme`
3. Make your changes and ensure all checks pass
4. Write a clear PR description explaining **what** and **why**
5. Reference any related issues (e.g., "Closes #42")

## Commit Convention

We use [Conventional Commits](https://www.conventionalcommits.org/).

**Format:** `type: short description`

**Types:** `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `style`

Examples:
```
feat: add sort-by-age option
fix: handle symlinks in scan
docs: add contributing guide
test: add scanner edge case tests
```

## Reporting Bugs

- Search existing issues first to avoid duplicates
- Include steps to reproduce, expected vs actual behavior, and your OS/Node version
- A minimal reproduction is extremely helpful

## Suggesting Features

- Open an issue with the `feature` label
- Describe the problem you're trying to solve, not just the solution
- We value simplicity — proposals that keep the tool focused are preferred

## AI-Generated Contributions

If your contribution was substantially generated by AI tools, please disclose this in your PR description. We welcome AI-assisted contributions but value transparency.

## Questions?

Open a [Discussion](https://github.com/westpoint-io/dustoff/discussions) or an issue tagged `question`. We're happy to help.
